'use strict', Object.freeze() y Object.seal() en JavaScript

En este documento, explicaremos el modo estricto ('use strict') y dos métodos útiles para controlar la mutabilidad de los objetos: Object.freeze() y Object.seal().

1. 'use strict'

'use strict' es una directiva que activa el modo estricto en JavaScript. Este modo ayuda a escribir código más seguro y a detectar errores comunes.

Ejemplo:

'use strict';

x = 3.14; // Esto lanzará un error porque x no está declarada

function ejemplo() {
  y = 5; // Esto también lanzará un error
}

ejemplo();

Beneficios del modo estricto:

  • Convierte errores silenciosos en errores que lanzan excepciones.
  • Corrige errores que hacen difícil para los motores de JavaScript realizar optimizaciones.
  • Prohíbe cierta sintaxis que probablemente será definida en futuras versiones de ECMAScript.

2. Object.freeze()

Object.freeze() congela un objeto: impide que se le agreguen nuevas propiedades, que se eliminen las existentes y que se modifiquen sus valores.

Ejemplo:

const persona = {
  nombre: "Ana",
  edad: 28
};

Object.freeze(persona);

persona.edad = 29; // Esto no tendrá efecto
persona.ciudad = "Madrid"; // Esto no agregará la propiedad

console.log(persona); // Imprime: { nombre: "Ana", edad: 28 }

3. Object.seal()

Object.seal() sella un objeto: impide que se agreguen o eliminen propiedades, pero permite modificar las propiedades existentes.

Ejemplo:

const coche = {
  marca: "Toyota",
  modelo: "Corolla"
};

Object.seal(coche);

coche.modelo = "Camry"; // Esto sí funciona
coche.año = 2021; // Esto no agregará la propiedad

console.log(coche); // Imprime: { marca: "Toyota", modelo: "Camry" }

4. Diferencias entre Object.freeze() y Object.seal()

Aunque ambos métodos restringen la modificación de objetos, tienen algunas diferencias clave:

  • Object.freeze():
    • No permite agregar nuevas propiedades.
    • No permite eliminar propiedades existentes.
    • No permite modificar los valores de las propiedades existentes.
  • Object.seal():
    • No permite agregar nuevas propiedades.
    • No permite eliminar propiedades existentes.
    • Permite modificar los valores de las propiedades existentes.

Ejemplo comparativo:

const objetoCongelado = { a: 1, b: 2 };
const objetoSellado = { x: 10, y: 20 };

Object.freeze(objetoCongelado);
Object.seal(objetoSellado);

objetoCongelado.a = 100; // No tiene efecto
objetoSellado.x = 100; // Sí tiene efecto

console.log(objetoCongelado); // Imprime: { a: 1, b: 2 }
console.log(objetoSellado); // Imprime: { x: 100, y: 20 }

Recuerda: Tanto Object.freeze() como Object.seal() son útiles para controlar la mutabilidad de los objetos, pero Object.freeze() es más restrictivo. Elige el método que mejor se adapte a tus necesidades de inmutabilidad.
